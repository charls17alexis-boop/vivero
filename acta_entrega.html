1. Declaración del Objeto

Por medio del presente documento formal, se hace constar que el desarrollador del software antes mencionado realiza la entrega física, lógica y operativa de la plataforma web denominada Sistema Integral de Gestión de Vivero (SIGV) en su versión estable 1.0.0. Dicho software ha sido desarrollado cubriendo estándares de diseño, arquitectura cliente-servidor y seguridad requeridos para la gestión integral de operaciones de un vivero.

El receptor del sistema manifiesta de conformidad haber recibido el código fuente, la base de datos estructurada, el acceso al superusuario administrador y la documentación correspondiente para su puesta en marcha.

2. Alcance del Sistema Entregado

A. Módulo de Autenticación y Control de Acceso (ACL)

  • Control de Sesiones con Token: Inicio de sesión mediante token criptográfico SHA-256. Cada petición a la API requiere el token en el encabezado x-auth-token.
  • Cifrado de Contraseñas: Algoritmo Bcrypt con saltos automáticos. Ninguna contraseña se almacena en texto plano.
  • Filtro de Seguridad por Roles (Middleware): Validación estricta a nivel de servidor (requireRole()). Si un vendedor intenta crear/editar recibe HTTP 403.
  • Prevención de Inyecciones SQL: Consultas parametrizadas en todas las operaciones de base de datos.
  • Validación de Contraseñas: Reglas estrictas (8+ caracteres, mayúscula, minúscula, número, símbolo).

B. Módulo de Dashboard

Resumen ejecutivo con tarjetas de métricas en tiempo real: Plantas en stock, Ventas del mes, Alertas de inventario, Clientes activos, Personal activo, Ventas de hoy, Gastos operativos estimados, Ticket promedio. Gráfico de barras de ventas por categoría, alertas de inventario codificadas por colores y ranking de top 5 productos más vendidos del mes.

C. Módulo de Usuarios

Gestión completa de cuentas: crear, editar, activar/desactivar y eliminar usuarios. Roles definidos: Administrador (acceso total) y Vendedor (acceso restringido a lectura, solo puede crear ventas). Cambio de contraseña con validación en tiempo real mediante checklist.

D. Módulo de Plantas

Catálogo completo con nombre, categoría, precio, stock, descripción y estado. Administrador: CRUD completo. Vendedor: visualización de solo lectura con etiqueta "Solo consulta".

E. Módulo de Clientes

Base de datos con nombre, teléfono, correo, dirección, RFC y CURP con validación de unicidad. Administrador: CRUD completo. Vendedor: solo lectura.

F. Módulo de Ventas

Registro de ventas con selección de cliente, método de pago (Efectivo, Tarjeta, Transferencia) y productos múltiples. Selector inteligente con stock disponible en tiempo real. Validación de stock en doble capa: frontend (UX inmediata) y backend (transacción ACID con rollback automático si algún producto no tiene suficiente existencia). Folio único automatizado (F- + número secuencial). Vendedor puede crear ventas pero no editar ni eliminar existentes.

G. Módulo de Inventario

Control de existencias con vista consolidada de plantas y herramientas. Tabla que muestra tipo, producto, cantidad actual, mínimo requerido y estado codificado por colores (Verde: suficiente, Amarillo: bajo, Rojo: agotado). Ajuste de stock solo para administrador con registro automático en historial de movimientos.

H. Módulo de Herramientas

Catálogo con nombre, tipo, cantidad, estado y ubicación. Administrador: CRUD completo. Vendedor: solo lectura.

I. Módulo de Facturación

Ventas por facturar con botón para generar CFDI. Procesamiento automático con folio fiscal (CFDI- + folio de venta) y captura de RFC del cliente. Historial completo de facturas emitidas con estado y fecha.

J. Módulo de Personal

Gestión de empleados del vivero con área (Campo, Invernadero, Tienda, Biofábrica, Oficina), puesto, teléfono y estado (Activo/Inactivo). Solo administrador.

K. Módulo de Biofábrica

Seguimiento de propagación con tipo de explante, medio de cultivo, fechas de siembra y trasplante, y estado. Solo administrador.

L. Módulo de Proveedores

Directorio con nombre, contacto, teléfono, correo y tipo de insumo. CRUD solo administrador.

M. Módulo de Calidad

Inspecciones con lote, inspector, fecha, estado fitosanitario, calificación y observaciones. Solo administrador.

N. Módulo de Reportes PDF

Generación de documento PDF profesional con resumen general, ventas por semana con tabla, productos más vendidos, clientes nuevos registrados y gastos operativos estimados.

4. Plan de Liberación

4.1 Requisitos para Ejecución Local

  • Instalar Node.js (versión 16 o superior) desde nodejs.org
  • Descomprimir el proyecto y abrir terminal en la carpeta
  • Ejecutar: npm install y luego npm start
  • Abrir navegador en http://localhost:3000

4.2 Usuarios por Defecto

4.3 Compartición en Red Local

Ejecutar npm start, verificar la IP local mostrada en la consola y abrir http://[IP]:3000 desde otras PCs en la misma red. El firewall de Windows debe permitir conexiones en el puerto 3000.
